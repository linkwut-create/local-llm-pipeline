# Local MCP — Model Context Protocol Integration

## Purpose

`local_llm_mcp_server.py` exposes the local LLM pipeline as MCP tools so Claude Code can invoke them natively without shelling out.

## Quick Check

```bash
# Verify the MCP server file exists and imports cleanly
python -c "import sys; sys.path.insert(0, 'tools'); import local_llm_mcp_server; print('OK:', len(local_llm_mcp_server.TOOLS), 'tools')"

# Run the health check tool via CLI for comparison
python tools/local_llm_check.py
```

## Available MCP Tools

| Tool | What it does |
|---|---|
| `local_check` | Environment health check |
| `local_summarize_file` | Summarize a file |
| `local_summarize_tree` | Summarize a directory tree |
| `local_generate_test_plan` | Generate test plan |
| `local_review_diff` | Single-model diff review |
| `local_debate_review_diff` | Multi-model debate review (fast+summary by default) |

## MCP Output is Advisory Only

All MCP tool output is generated by **local models** that:
- Cannot run tests
- Cannot read files outside the input
- Cannot modify code
- Have limited project context

**Claude Code must verify all important claims.** MCP output is a candidate finding, not a final judgment.

## When to Use MCP vs CLI

- **MCP** for routine diff reviews, summaries, test plans during coding sessions
- **CLI** for debugging, first-time setup, or when you need raw error output

## Security

- All MCP tools are read-only
- No file writing, no shell execution, no git operations
- Blocked paths enforced at MCP boundary
- Subprocess timeout prevents hung tool calls
